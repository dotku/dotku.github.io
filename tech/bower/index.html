Bower

Introduction

Bower is a front-end package manager tool provided by Twitter since 2012, and driven by the community in GitHub.

Common Usage

You can use command like this:

bower install bootstrap

Well, you can install a specified version with this command:

bower install jquery#1.9.0

You might want to know available version for a package, you can use this:

bower info jquery

It will return the report like this:

C:\>bower info jquery
bower cached        git://github.com/jquery/jquery.git#2.1.4
bower validate      2.1.4 against git://github.com/jquery/jquery.git#*

{
  name: 'jquery',
  version: '2.1.4',
  main: 'dist/jquery.js',
  license: 'MIT',
...
}

Available versions:
  - 2.1.4
  - 2.1.3
  - 2.1.2
  - 2.1.1
...
  - 1.0.1

Show 19 additional prereleases with ‘bower info jquery --verbose’
You can request info for a specific version with 'bower info jquery#<version>'

In bower, the target install directory would be the package name. You might want to install a package in different version, you can install it by this:

bower install targetDirectory=packageName#version

Example:

bower install jquery-1.9.0=jquery#1.9.0

Config

By default, the target directory will be bower_components, if you don't like the the setting you can place a .bowerrc under your project directory. If you want to make global, just place it to the root, for example C:/ in Windows, or / in Linux/Unix.

Enjoy the bower :)

More: http://www.bower.io/
